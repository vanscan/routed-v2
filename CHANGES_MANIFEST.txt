WHY THIS ZIP EXISTS
-------------------
The managed "Save to GitHub" action fails with a non-fast-forward (500) because
the workspace `main` has diverged from GitHub's `main`. This zip contains the

you can upload them to GitHub manually and make the remote match the workspace.

HOW TO UPLOAD
-------------
Option A (GitHub web UI): open your repo → "Add file" → "Upload files" → drag the
  folders from this zip (keep the same paths) → commit to `main`.
Option B (git CLI on your machine):
  1) Clone/pull your repo.
  2) Copy these files over the same paths.
  3) git add -A && git commit -m "Native map migration Phase 0-3 + OSRM fix" && git push

NOTE: paths in this zip are relative to the repo root (e.g. frontend/..., backend/...).
Do NOT upload node_modules, .expo, caches, or .env files.
